Goals
1) Code hygiene and structure
   - Convert ad-hoc HTML into a consistent component-driven structure.
   - Create reusable components: Header, Footer, Hero, Section, FAQ, CTA buttons, Card, Badge, FeatureList.
   - Extract shared colors, spacing, and typography tokens into a single theme file (CSS variables or Tailwind config).
   - Normalize file naming: kebab-case for files, PascalCase for components.

2) Styling
   - Use a single styling system (Tailwind or CSS Modules). Remove duplicate/global ad-hoc CSS.
   - Ensure mobile-first responsive rules. Verify headings and spacing scales on 320px, 768px, 1024px widths.
   - Add focus-visible styles for all interactive elements.

3) Accessibility
   - Semantic HTML for sections and headings (one h1 per page).
   - Button vs link correctness, aria-labels for icons, alt text for images.
   - Keyboard nav and skip-to-content link.
   - Color-contrast AA minimum.

4) Performance and UX
   - Optimize images and use width/height attributes to prevent CLS.
   - Defer non-critical scripts, preload critical fonts, and use font-display: swap.
   - Remove unused dependencies and dead code.
   - Lighthouse targets: Performance 90+, Accessibility 95+, SEO 95+.

5) Content consistency
   - Centralize CTA variants (“Get my options”, “Get Started”, “Start assessment”) into a single component with props.
   - Ensure internal links use consistent paths and that external links open with rel=\"noopener\".
   - Create a shared FAQ component and drive Q&A from a simple JSON array.

6) SEO basics
   - Add <title>, <meta name=\"description\">, canonical, and Open Graph tags per page.
   - Logical h1/h2 structure, meaningful link text, and unique descriptions.
   - Generate a sitemap.xml and robots.txt.

7) Build & quality gates
   - Add Prettier + ESLint with a reasonable config (airbnb or standard). Add format and lint scripts.
   - TypeScript optional: if present, strengthen types for props and utilities.
   - Add a minimal CI step (if supported) to run lint and build.
